Setup Guide

Before you run oc-setup, you need three things: your Telegram user ID, a Telegram bot token, and an API key from an AI provider. This guide walks you through getting each one.

Requirements

  • Node.js 18 or later — check with node -v. If you don't have it, oc-setup will tell you how to install it.
  • A Telegram account — for creating your bot and getting your user ID.
  • An API key — from any supported provider (Anthropic, OpenAI, Google, xAI, DeepSeek, Mistral, Perplexity, Kimi, Groq, or OpenRouter).

Finding Your Telegram User ID

Your user ID is a number that identifies your Telegram account. OpenClaw needs it to make sure only you can talk to your bot.

  1. Open Telegram
  2. Search for @userinfobot and start a chat
  3. Send any message — it will reply with your user ID
  4. Copy the number (e.g. 123456789)

This ID gets added to your OpenClaw config as an "allowed sender" so only you can message your bot. See Allowed Senders for more details.

Creating a Telegram Bot Token

  1. Open Telegram and search for @BotFather
  2. Send /newbot
  3. Choose a display name for your bot (e.g. "John's AI")
  4. Choose a username ending in bot (e.g. johns_ai_bot)
  5. BotFather will send you a token like 7891234567:AAH...
  6. Copy the full token
Keep your bot token secret. Anyone with it can control your bot. If it leaks, message @BotFather and send /revoke to get a new one.

oc-setup / oc-setup init

The main command. Runs the full guided setup flow:

  1. Detects your OS and verifies Node.js
  2. Installs or updates OpenClaw globally
  3. Prompts for your Telegram bot token and validates it
  4. Prompts for your AI provider API key and validates it
  5. Generates SOUL.md, USER.md, and HEARTBEAT.md from your preferences
  6. Starts the gateway and configures auto-start on boot
  7. Runs verification to confirm everything is working

Idempotent — safe to run again if something failed. It skips steps that are already complete.

Quick Setup (skip prompts)

Use the Quick Setup form on the homepage to generate a command with everything pre-filled:

npx oc-setup --config <base64-payload>

oc-setup doctor

Runs health checks and tells you exactly what's wrong and how to fix it:

oc-setup doctor

 ✓ Node.js      v22.1.0 (OK)
 ✓ OpenClaw     v2026.3.2 (OK)
 ✗ Gateway      NOT RUNNING
   → Fix: openclaw gateway start
 ✓ Telegram     connected
 ✗ API key      expired
   → Fix: openclaw config set model.apiKey "sk-ant-new..."
 ✓ Disk         42GB free (OK)

Checks: Node.js version, OpenClaw installed, gateway running, Telegram connected, API key valid, disk space, config file, auto-start service.

oc-setup update

Updates OpenClaw to the latest version. Preserves your config and restarts the gateway automatically.

oc-setup reset

Nuclear option. Backs up your current config to ~/.openclaw-backup-<timestamp>/, wipes everything, and re-runs the full setup from scratch.

oc-setup add <channel>

Add additional messaging channels. Coming in v2:

  • oc-setup add discord
  • oc-setup add imessage
  • oc-setup add whatsapp

Allowed Senders

OpenClaw uses your Telegram user ID to restrict who can talk to your bot. Without this, anyone who finds your bot could send it commands.

During setup, your user ID is added to the allowFrom list in ~/.openclaw/openclaw.json:

{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "your-bot-token",
      "dmPolicy": "allowlist",
      "allowFrom": ["123456789"]
    }
  }
}

Only user IDs in allowFrom can message your bot. Everyone else is ignored.

Adding more allowed senders

To let another person talk to your bot:

openclaw config set channels.telegram.allowFrom '["123456789", "987654321"]'

Or edit ~/.openclaw/openclaw.json directly and restart the gateway with openclaw gateway restart.

Installing Skills

Skills are extensions that teach your agent new capabilities. Browse 3,000+ skills at clawhub.com.

Install the ClawHub CLI

npm install -g clawhub

Common commands

clawhub search "calendar"       # search for skills
clawhub install gog              # install a skill
clawhub update --all             # update all installed skills

Changing Models

To switch your AI model after setup, edit your config:

openclaw config set model.provider "anthropic"
openclaw config set model.modelId "claude-opus-4-6"
openclaw gateway restart

Or edit ~/.openclaw/openclaw.json directly under the model section.

Common Issues

Gateway won't start

openclaw gateway start
# If port is in use:
lsof -i :18789
kill <PID>
openclaw gateway start

Telegram bot not responding

  • Check the token is valid: oc-setup doctor
  • Make sure your user ID is in allowFrom
  • Restart the gateway: openclaw gateway restart

API key expired or invalid

openclaw config set model.apiKey "your-new-key"
openclaw gateway restart

Node.js too old

OpenClaw requires Node.js 18+. Update with:

# macOS
brew install node

# Linux
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

"openclaw: command not found"

npm install -g openclaw

Uninstalling

Quick uninstall (keeps your data)

npm uninstall -g openclaw
npm uninstall -g oc-setup

Full uninstall (removes everything)

1. Stop the gateway and remove auto-start:

macOS:

openclaw gateway stop
launchctl unload ~/Library/LaunchAgents/com.openclaw.gateway.plist
rm ~/Library/LaunchAgents/com.openclaw.gateway.plist

Linux:

openclaw gateway stop
systemctl --user disable openclaw-gateway.service
rm ~/.config/systemd/user/openclaw-gateway.service
systemctl --user daemon-reload

2. Remove packages:

npm uninstall -g openclaw
npm uninstall -g oc-setup

3. Remove all data and config:

rm -rf ~/.openclaw
This deletes your config, SOUL.md, USER.md, HEARTBEAT.md, logs, and all agent memory. Back up ~/.openclaw first if you want to keep anything.
